# Supabase + Backend Integration Guide

## Overview

Complete Supabase schema + Node.js/Express backend code for Signal app.

**Files provided:**
- `SUPABASE_SETUP.sql` — Complete database schema (8 tables, RLS, triggers, views)
- `server/routes/agencies.js` — Agency management endpoints
- `server/routes/watchlist.js` — Watchlist & alerts endpoints
- `server/routes/groups.js` — Group scans endpoints

---

## 1. Set Up Supabase Database

### Step 1: Copy SQL

Open your Supabase project → SQL Editor → New Query

Copy the entire content from `SUPABASE_SETUP.sql` and run it.

**What gets created:**
- `agencies` — User's talent agencies
- `agency_creators` — Creators from agencies (with niche, platform, follower count)
- `creator_watchlist` — Tracked creators with priority/notes
- `creator_alerts` — Post/deal alerts (generated by background jobs)
- `creator_groups` — Named groups for batch scanning
- `group_creators` — M2M relationship (creators in groups)
- `scan_jobs` — Background scan job tracking
- `user_settings` — User preferences

**Automatic features:**
- Row Level Security (RLS) — Users can only see their own data
- Indexes on common queries
- Timestamp triggers (auto-updated `updated_at`)
- Foreign key cascades (delete agency → delete creators)

### Step 2: Verify RLS is Enabled

In Supabase dashboard:
- Go to each table
- Auth → Policies should show RLS policies for each
- Green checkmark = enabled ✓

### Step 3: Get Your Credentials

Supabase dashboard → Settings → API

Copy:
- **Project URL** → `SUPABASE_URL`
- **Anon Key** → `SUPABASE_ANON_KEY` (public, safe for browser)
- **Service Role Key** → `SUPABASE_SERVICE_ROLE_KEY` (secret, for backend only!)

## 2. Set Up Backend Routes

### Step 1: Create Route Files

Copy these files into your `server/routes/` directory:
- `agencies.js` — GET/POST/DELETE agencies, manage creators
- `watchlist.js` — Watchlist, alerts management
- `groups.js` — Creator groups, scan jobs

```bash

```

### Step 3: Configure Environment

Add to `.env`:

```

SUPABASE_ANON_KEY=eyJ0eXAiOiJKV1QiLCJhbGc...
SUPABASE_SERVICE_ROLE_KEY=eyJ0eXAiOiJKV1QiLCJhbGc...
```

In your main `server.js`:

```javascript

const groupsRouter = require('./routes/groups');

app.use('/api/agencies', agenciesRouter);
app.use('/api/watchlist', watchlistRouter);

```

```bash
curl -X GET http://localhost:3000/api/agencies \
  -H "Authorization: Bearer YOUR_JWT_TOKEN"
```

Should return `[]` (empty array) if authenticated correctly.

## 3. Frontend Integration

### Get JWT Token from Supabase Auth

```javascript
const { data, error } = await supabase.auth.signInWithPassword({
  email: 'user@example.com',
  password: 'password',

const session = data.session;

```

### Make API Calls

```javascript
// Add agency
const response = await fetch('/api/agencies', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
    'Authorization': `Bearer ${token}`,
  },
  body: JSON.stringify({
    name: 'CAA Talent',
    url: 'https://caa.com',
    description: 'Hollywood talent agency',
  }),

const agency = await response.json();
```

### Or Use Fetch Wrapper

```javascript
class API {
  constructor(token) {
    this.token = token;
  }

  async request(endpoint, options = {}) {
    const response = await fetch(`/api${endpoint}`, {
      ...options,
      headers: {
        'Content-Type': 'application/json',
        'Authorization': `Bearer ${this.token}`,
        ...options.headers,
      },
    });

    if (!response.ok) {
      throw new Error(`API error: ${response.statusText}`);
    }

    return response.json();
  }

  // Agencies
  getAgencies() {
    return this.request('/agencies');
  }

  createAgency(data) {
    return this.request('/agencies', {
      method: 'POST',
      body: JSON.stringify(data),
    });
  }

  deleteAgency(id) {
    return this.request(`/agencies/${id}`, { method: 'DELETE' });
  }

  // Watchlist
  getWatchlist() {
    return this.request('/watchlist');
  }

  addToWatchlist(creatorId, notes) {
    return this.request('/watchlist', {
      method: 'POST',
      body: JSON.stringify({ creator_id: creatorId, notes }),
    });
  }

  // Alerts
  getAlerts(options = {}) {
    const params = new URLSearchParams(options).toString();
    return this.request(`/alerts?${params}`);
  }

  // Groups
  getGroups() {
    return this.request('/groups');
  }

  createGroup(name, creatorIds = []) {
    return this.request('/groups', {
      method: 'POST',
      body: JSON.stringify({ name, creator_ids: creatorIds }),
    });
  }

  startScan(groupId) {
    return this.request(`/groups/${groupId}/scan`, { method: 'POST' });
  }
}

// Usage
const token = session.access_token;
const api = new API(token);

const agencies = await api.getAgencies();
const watchlist = await api.getWatchlist();
```

## 5. Authentication Flow

### Get Token

```javascript
// User logs in
const { data, error } = await supabase.auth.signInWithPassword({
  email,
  password,
});

if (error) throw error;

const token = data.session.access_token;
localStorage.setItem('token', token);
```

### Attach Token to Requests

All API calls need the JWT token in header:

```javascript
headers: {
  'Authorization': `Bearer ${token}`,
}
```

### Verify Token on Backend

Routes automatically verify the token:

```javascript
async function verifyAuth(req, res, next) {
  const token = req.headers.authorization?.split(' ')[1];
  
  // Verify with Supabase
  const { data, error } = await supabase.auth.getUser(token);
  
  req.user = data.user;  // Now available in route handlers
  next();
}
```

## 6. Row Level Security (RLS)

All tables have RLS policies that automatically enforce user isolation:

```sql
-- Example: Users can only see their own agencies
CREATE POLICY "Users can view their own agencies"
  ON public.agencies
  FOR SELECT
  USING (auth.uid() = user_id);
```

**What this means:**
- No need to filter by `user_id` in queries
- Database automatically filters
- Impossible for user A to access user B's data (even with SQL injection)

## 7. Common Patterns

### Create Agency + Add Creators (Batch)

```javascript
// 1. Create agency
const agency = await api.request('/agencies', {
  method: 'POST',
  body: JSON.stringify({
    name: 'CAA',
    url: 'https://caa.com',
  }),
});

// 2. Add creators to agency
const creators = [
  { handle: '@creator1', name: 'Creator One', niche: 'FPS / Competitive' },
  { handle: '@creator2', name: 'Creator Two', niche: 'Fortnite / Battle Royale' },
];

const result = await api.request(`/agencies/${agency.id}/creators/batch`, {
  method: 'POST',
  body: JSON.stringify({ creators }),
});
```

### Create Group + Start Scan

```javascript
// 1. Create group
const group = await api.request('/groups', {
  method: 'POST',
  body: JSON.stringify({
    name: 'Q2 Deals - Fortnite Creators',
    creator_ids: ['creator-id-1', 'creator-id-2', ...],
  }),
});

// 2. Start scan
const job = await api.request(`/groups/${group.id}/scan`, {
  method: 'POST',
});

// 3. Poll for status
const pollStatus = setInterval(async () => {
  const status = await api.request(`/groups/${group.id}/scan`);
  
  if (status.job_status === 'completed') {
    clearInterval(pollStatus);
    console.log(`Scan complete! Found ${status.alerts_created} alerts`);
  }
}, 2000);
```

### Watch Creator + Get Alerts

```javascript
// 1. Add to watchlist
await api.request('/watchlist', {
  method: 'POST',
  body: JSON.stringify({
    creator_id: 'creator-id',
    notes: 'High priority - big followership',
    priority: 1,  // 0=normal, 1=high, -1=low
  }),
});

// 2. Get unread alerts
const alerts = await api.request('/alerts?read=false');

// 3. Mark as read
await api.request(`/alerts/${alerts[0].id}`, {
  method: 'PATCH',
  body: JSON.stringify({ read: true }),
```

## 8. Background Jobs (Scan Processing)

The `/api/groups/:id/scan` endpoint queues a job. You need to implement:

### Option A: Bull Queue (Recommended)

```bash
npm install bull redis
```

```javascript
// server/lib/queue.js
const Queue = require('bull');

const scanQueue = new Queue('group-scans', {
  redis: {
    host: process.env.REDIS_HOST,
    port: process.env.REDIS_PORT,
  },
});

// Process scan jobs
scanQueue.process(async (job) => {
  const { jobId, userId, groupId } = job.data;

  try {
    // Mark as running
    await supabase
      .from('scan_jobs')
      .update({ job_status: 'running', started_at: new Date() })
      .eq('id', jobId);

    // Get creators in group
    const { data: creators } = await supabase
      .from('group_creators')
      .select('creator:agency_creators(*)')
      .eq('group_id', groupId);

    // Process each creator (fetch posts, detect deals, etc.)
    let alertsCreated = 0;
    for (const { creator } of creators) {
      // Your scanning logic here
      // Create alerts when deals detected
      alertsCreated += await scanCreator(userId, creator);
    }

    // Mark as completed
    await supabase
      .from('scan_jobs')
      .update({
        job_status: 'completed',
        alerts_created: alertsCreated,
        completed_at: new Date(),
      })
      .eq('id', jobId);

    return { jobId, alertsCreated };
  } catch (err) {
    // Mark as failed
    await supabase
      .from('scan_jobs')
      .update({
        job_status: 'failed',
        error_message: err.message,
        completed_at: new Date(),
      })
      .eq('id', jobId);

    throw err;
  }
});

module.exports = { queue: scanQueue };
```

### Option B: Simple Cron Jobs

If you don't have Redis, use a simple interval:

```javascript
// Process pending scans every 10 seconds
setInterval(async () => {
  const { data: jobs } = await supabase
    .from('scan_jobs')
    .select('*')
    .eq('job_status', 'queued')
    .limit(5);

  for (const job of jobs) {
    await processScanJob(job);
  }
}, 10000);
```

## 11. Troubleshooting

### "Missing authorization token"
- Make sure you're sending the JWT in `Authorization: Bearer` header
- Check token hasn't expired (`exp` claim in JWT)

### "User not found"
- Supabase can't verify the token
- Check `SUPABASE_SERVICE_ROLE_KEY` is correct
- Token might be old — re-login

### "Invalid RLS policy"
- You tried to insert without `user_id`
- Or you're accessing data for wrong user (RLS blocking it)
- Check the user_id matches `auth.uid()`

### Scan job never completes
- Implement background job processing (Bull queue recommended)
- Check Redis connection
- Monitor job queue for errors
